第一次 `code status` 显示 `docs/journal` 为 Clean，但远程实际有新 commit。可能 fetch 后 `remote_head` 没有正确参与比对。手动 `git merge --ff-only origin/main` 拉取。

## fetch 作用域分析

`code status` 的 fetch 逻辑在 `src/commands/code.rs:152-161`：

```rust
if let Ok(repo) = git2::Repository::open(&self.root) {
    if let Ok(mut remote) = repo.find_remote("origin") {
        remote.fetch(&["+refs/heads/*:refs/remotes/origin/*"], ...);
    }
}
```

问题是：fetch 发生在**父仓库**，更新的是父仓库的 remote refs。而 `remote_head` 来自**子模块仓库**自身的 remote refs。父仓库的 fetch 不会传播到子模块。

所以第一次 `code status` 显示 Clean 不是 fetch 没执行，而是 fetch 执行在了错误的作用域。

`RepoState::scan()` 读取的是子模块自己的 refs/remotes/origin/*，这部分没有被刷新。
